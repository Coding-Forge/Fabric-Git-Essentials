Fabric + Git Workshop Essentials
Power BI DevOps Enablement
A visual summary of key insights, best practices, and hands-on labs from the Fabric + Git integration workshop. Covers CI/CD for both Azure DevOps and GitHub Actions.

Key Takeaways

  • PBIP Format: Enables version control of Power BI assets via Git.
  • CI/CD Pipelines: Automate validation and deployment using Azure DevOps or GitHub Actions.
  • Sparse Checkout: Clone only the folders you need — platform presets for AzDO or GitHub.
  • Universal Pipeline: One shared template repo consumed by many Fabric project repos.
  • Governance: Enforce RLS/CLS, workspace roles, and naming conventions.
  • Embedded Analytics: Use service principals for secure Power BI embedding.
  • Security: Automated secret scanning on every push and PR.

Governance Best Practices

  • Use Dev/Test/Prod workspace separation
  • Define RBAC roles and RACI matrix
  • Adopt naming conventions for items
  • Implement PR templates with checklists
  • Enforce RLS/CLS and sensitivity labels
  • Use deployment pipelines with approvals

Embedded Analytics Setup

  • Register Azure AD App
  • Enable service principal in Power BI Admin
  • Assign workspace roles to app
  • Use Embed API with service principal
  • Validate RLS and performance
